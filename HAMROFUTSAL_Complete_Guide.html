Line 15: This tells PDO: "If any SQL error happens, throw an Exception (a loud error) so I can catch and debug it." Without this, errors would be silent and your code would break with no warning. Always keep this ON during development!

Line 18: When we fetch data from the database, this tells PDO to return results as associative arrays. That means instead of accessing data like $row[0] (confusing), we can use $row['name'] (clear and readable). Much easier for beginners!

23}
Lines 19-23: If the connection FAILS (maybe MySQL is not running in XAMPP), this catch block catches the error. die() stops the entire page and shows the error message. Common fix: Open XAMPP Control Panel and make sure MySQL is running (green light)!
💡 Student Tip: This file is included by every page using require_once 'includes/db.php';. The variable $conn becomes available globally for all database queries on that page.

📄 schema.sql

database/schema.sql
🎯 What does this file do? This defines ALL the tables in our database. Think of tables like Excel spreadsheets — each table stores a specific type of data. This project has 6 tables.
Table Name What It Stores Key Columns
users All user accounts (players + owners) id, name, email, password (hashed), role, phone
owner_details Extra business info for owners owner_id, business_name, pan_number, bank_details
futsals Futsal court listings id, owner_id, name, location, price_per_hour, image_path
reward_points Player loyalty points balance player_id, points (starts at 100)
bookings All slot reservations player_id, futsal_id, booking_date, start_time, end_time, status
reviews Player ratings & feedback for courts futsal_id, user_name, rating (1-5), message
Important SQL concepts used:
• AUTO_INCREMENT → ID numbers increase automatically (1, 2, 3...)
• PRIMARY KEY → The unique identifier for each row
• FOREIGN KEY → Links one table to another (e.g., bookings links to futsals via futsal_id)
• ON DELETE CASCADE → If a user is deleted, all their bookings/reviews are automatically deleted too
• ENUM('pending','approved','rejected','cancelled') → A column that can ONLY have these 4 values
• DEFAULT CURRENT_TIMESTAMP → Automatically saves the current date/time when a row is created
How tables connect to each other:

users (id) ← owner_details (owner_id)
users (id) ← futsals (owner_id)
users (id) ← reward_points (player_id)
users (id) ← bookings (player_id)
futsals (id) ← bookings (futsal_id)
futsals (id) ← reviews (futsal_id)

How the Reward System Works (in bookings.php):

When an owner approves a booking, the system:
1. Starts a database transaction (all-or-nothing)
2. Updates the booking status to 'approved'
3. Awards +15 reward points to the player
4. Checks if the player has completed a multiple of 5 bookings → if yes, gives an extra +20 bonus points!
5. Commits all changes together

When a player cancels within 6 hours of the scheduled time, they lose -20 points as a penalty.
Booking Approval & Reward Points Flow:

Player books a slot (status = 'pending')
↓
Owner sees booking in bookings.php → Clicks "Approve"
↓
PHP starts transaction → Updates status to 'approved'
↓
Awards +15 points to player → Checks 5th booking bonus (+20)
↓
Commits transaction → Player sees updated points

📊 Full Data Flow Diagram

Step 1: User Opens Website
→ Browser loads index.php → header.php runs → session_start() → CSS loaded

Step 2: User Registers
→ register.php → Validates input → password_hash() → INSERT INTO users → INSERT INTO reward_points (100 pts) → Redirect to login.php

Step 3: User Logs In
→ login.php → SELECT user from database → password_verify() → Save to $_SESSION → Redirect to dashboard

Step 4: Player Searches & Books
→ search.php → SELECT futsals with filters → Display cards → Click "Book Court" → book.php
→ JavaScript calls check_slots.php (AJAX) → Returns JSON of booked slots
→ Player picks date + time → POST form → INSERT INTO bookings (status: pending)

Step 5: Owner Approves Booking
→ owner/bookings.php → Shows pending bookings → Click "Approve"
→ BEGIN TRANSACTION → UPDATE booking status → UPDATE reward_points (+15) → Check 5th bonus (+20) → COMMIT

Step 6: Player Writes Review
→ book.php review form → INSERT INTO reviews → Display with star ratings

Step 7: User Logs Out
→ logout.php → $_SESSION = [] → session_destroy() → Redirect to index.php
🎓 Key Takeaways for Students:
• Always use PDO prepared statements for database queries (prevents SQL injection)
• Always use password_hash() and password_verify() (never store plain passwords)
• Always sanitize() user input before displaying it (prevents XSS attacks)
• Use sessions to remember logged-in users across pages
• Use transactions when modifying multiple tables together
• Use AJAX (fetch) for real-time updates without page refresh
• Use CSS variables for consistent design across the entire project
• Always call exit; after header("Location: ...") redirects
